🔍 WHY：为什么要写这篇论文？(痛点)

现有 RL 策略网络在引入机器人结构先验时，仅利用了连杆拓扑（link connectivity）进行消息传递，而动力学传播特性（inertia propagation）——即力与运动如何沿关节树级联传递——这一物理本质被完全忽视。

  • GNN 方法的不足：虽指定了信息交换的空间路径，但特征聚合的语义完全由数据驱动学习，缺乏物理约束，导致样本效率低、对动力学参数变化（如质量偏移）泛化差。
  • Transformer 方法的代价：全注意力机制参数量大，且无法显式编码树形层级结构，在关节自由度较多的人形机器人上尤为低效。
  • 核心问题：Articulated Body Algorithm (ABA) 是刚体正向动力学的标准计算流程，其惯性从叶到根聚合的计算结构天然适合作为策略网络的归纳偏置，但从未被系统性地嵌入到策略学习中。

💡 WHAT：它提出了什么？(本质)

ABD-Net (Articulated-Body Dynamics Network)：一种以 ABA 正向动力学计算结构为蓝本的图神经网络策略架构。

核心思想：用可学习参数替换 ABA 中的物理量，保留其从子连杆到父连杆的惯性聚合机制，使策略网络天然捕捉"动作如何通过身体传播"的动力学语义。

  • 每个连杆 i 拥有可学习参数：B_i ∈ ℝ^d（类比刚体惯性）和 W_i ∈ ℝ^{d×d}（类比运动子空间）
  • 信息传递严格遵循叶→根的树遍历顺序，模拟惯性向根部累积的物理过程
  • 总参数量仅 ~91–95K，与 MLP/GNN 基线持平

🛠️ HOW：它是如何实现的？(机理)

网络结构：三模块流水线 π_θ = Ψ ∘ M ∘ Φ

  1. 观测编码器 Φ：逐连杆投影 {φ_i}，将状态 s 映射为连杆嵌入 {z_i} ∈ ℝ^{K×d}
  2. 动力学消息传递 M（核心）：
    • 子连杆贡献：v_j^a = v_j − v_j · (W_j W_j^T v_j)——学习到的投影近似 ABA 的约束消除，识别传播中应衰减的特征方向
    • 连杆表征聚合：v_i = softplus(z_i + B_i) + Σ_{j∈ch(i)} v_j^a——softplus 保证正性，反映惯性的正定约束
    • 严格按叶→根顺序遍历（非并行，是设计选择）
  3. 动作解码器 Ψ：逐关节头 {ψ_j}，关节 j 的动作使用其父连杆表征 a_j = ψ_j(v_{pa(j)})

正交性约束（关键 Trick）

L_orth = (1/K) Σ_i ‖W_i^T diag(v_i) W_i − I‖²_F

正则化确保 (W^T diag(v) W)^{-1} ≈ I，防止数值不稳定的同时允许任务自适应。消融实验证明此约束对动力学泛化至关重要（Humanoid 质量泛化：91.1% → 59.8%）。

训练设置

  • RL 算法：PPO，总损失 = PPO 目标 + 正交约束
  • 仿真平台：Genesis（G1/Go1/Go2/T1）+ SAPIEN（Humanoid/Hopper），控制 50 Hz，仿真 200 Hz
  • 观测空间：关节位置/速度、IMU 角速度、投影重力、速度指令、前序动作、基座线速度；G1/T1 额外使用步态相位编码和足端位置
  • Reward 设计（Genesis）：
    • 线速度跟踪：权重 1.0–1.5
    • 基座高度正则化：权重 10.0–50.0（关键稳定项）
    • 动作变化率惩罚：0.005–0.1
    • 力矩惩罚：1e-5
    • 关节位置限位惩罚：5.0

Sim-to-Real 转移

  • 硬件：Unitree G1（人形）+ Go2（四足），NVIDIA Jetson Orin NX 板载计算
  • 推理延迟：<5 ms（G1），远低于 20 ms 控制周期预算
  • 域随机化：足底摩擦、编码器偏置、质心偏移、外力扰动
  • 输出：期望关节位置（相对默认站姿的偏移），PD 控制器 200 Hz 执行

⚠️ CRITICAL：局限与挑战 (反思)

  1. 计算瓶颈：叶→根的顺序遍历无法并行化，PyTorch 下训练耗时比 MLP 高 4–6×（JAX 下约 2×）。推理端可接受（<5 ms），但大规模训练受限。
  2. 仅支持本体感知：当前架构不处理高维感知输入（如视觉），限制了在导航/操作任务中的适用性。
  3. 任务范围窄：仅在运动控制上验证，操作（manipulation）和接触丰富任务未探索。
  4. 树结构假设：闭链机构（如平行四连杆）不在当前框架覆盖范围内。
  5. 物理先验的刚性：虽然参数可学，但叶→根单向传递结构是否足以覆盖所有需要双向信息流的任务场景（如操作中需要从末端执行器向基座传递反馈），值得探讨。

💎 与 OmniBot/RAL 的关联

  • 轮足机器人的结构先验：轮足机器人（如 OmniBot）具有明确的树形运动链，ABD-Net 的惯性传播机制可直接适配。轮式模块可视为叶节点，其惯性特性通过连杆聚合到躯干，为全身协调运动提供动力学一致的表征。
  • 动力学泛化对轮足的意义：轮足机器人在不同地形（如斜坡、台阶边缘）等效动力学参数会显著变化，ABD-Net 展现的质量偏移鲁棒性（平均 23.9% 提升）对这类场景极具价值。
  • 正交约束的可迁移性：正交约束作为一种轻量但高效的正则化手段，可直接嵌入任何基于 GNN 的策略网络，包括 RAL 论文中可能使用的图消息传递架构。
  • 实时推理的参考价值：<5 ms 推理延迟证明该架构可部署在 Jetson 级嵌入式平台上，与 OmniBot 的硬件约束一致。
  • 局限性提示：叶→根单向传递可能不足以描述轮足机器人的轮-地接触反馈回路，可能需要增加根→叶的反向传递通道（类似 ABA 的第二遍扫描）。
  • Reward 设计参考：基座高度正则化权重极高（10–50），暗示在腿式机器人运动控制中，姿态稳定性的奖励权重需要远高于跟踪精度，这对 RAL 的 reward 调参有直接参考意义。
